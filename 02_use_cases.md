# Use Case 1 (см. файл пункт 4.3.1)
**Название**: Импорт расписания через Excel-файл.

**Роль**: Староста.

**Предусловие**:
1. Создана база данных.
2. Староста зарегистрирован в боте (вошел как староста отпределенной группы).

**Основной поток**:
1. Староста вводит команду /import_schedule для импорта расписания в базу данных.
2.  Система выводит три способа (Excel-файл, фото, ввести вручную) загрузки расписания.
3. Староста выбирает для загрузки данных Excel-файл. 
4. Система предлагает скачать шаблон для заполнения расписания.
5. Староста вводит команду /template.
6. Система отправляет Excel-файл schedule_template.xlsx.
7. Староста скачивает, заполняет файл и отправляет его обратно в систему.
8. Система обрабатывает полученные данные, выводит их пользователю для проверки. Спрашивает все ли верно.
9. Староста, если все верно, вводит команду /confirm_schedule.
10. Система сохраняет данные и оповещает об этом пользователей.

**Альтернативный поток**:
1. Система вывела в 8 пункте основного потока данные с ошибками.
1.1 Староста вводит команду /edit_schedule.
1.2 Система отправляет пользователю файл (с ошибками).
1.3 Староста скачивает себе файл, исправляет ошибки, загружает исправленный файл в систему.
   
**Постусловие**:
1. В системе загружено расписание.
2. Группа получила уведомление о загруженном расписании. 

***В этом Use Case под системой подразумевается бот NAKOL.***
