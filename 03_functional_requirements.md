# FR-1: Регистрация и роли

FR-1.1 Система должна запускать процесс регистрации при получении команды `/start` от пользователя, который не присутствует в хранилище зарегистрированных пользователей.

FR-1.2 Система должна предложить пользователю выбрать одну из следующих ролей:  
- Староста  
- Индивидуальный пользователь  
- Участник группы  

FR-1.3 Система должна принять выбор, только если пользователь явно выбрал один из трех предложенных вариантов. Если выбор не соответствует ни одному из вариантов, система должна отклонить его и повторить запрос.

FR-1.4 После успешного выбора роли система обязана сохранить следующие данные в постоянном хранилище SQLite:  
- Уникальный идентификатор пользователя `user_id`  
- Выбранную роль `role`   
- Уникальный идентификатор пользователя в Telegram `telegram_id`  
- Идентификатор группы (для starosta и group_member) или NULL (для individual) `group_id`  

FR-1.5 Система должна отправить пользователю сообщение, подтверждающее успешную регистрацию и содержащее назначенную ему роль или данные, ссответствующие его роли.

# FR-2: Управление группами

FR-2.1 Система должна предоставить пользователю с ролью `starosta` возможность создать группу через команду `/create_group`.

FR-2.2 При создании группы система должна сгенерировать уникальный код доступа и сохранить группу в базе данных ( код группы `group_code`, имя группы `group_name`, id старосты `starosta_id`), а также направить код старосте.

FR-2.3 Система должна предоставить пользователю с ролью `group_member` возможность присоединиться к группе при выборе роли, указав код доступа.

FR-2.4 Пользователи с ролью индивидуал не должны иметь доступа к командам управления группами.

# FR-3: Импорт расписания

FR-3.1 Система должна предоставить пользователям с ролью староста или индивидуал возможность импортировать расписание через команду `/import_schedule`.

FR-3.2 Система должна предложить три способа импорта:
* Загрузить Excel-файл 
* Отправить фото (OCR)
* Ввести вручную 

FR-3.3 После загрузки расписания любым способом система должна отобразить пользователю список распознанных/введённых занятий и запросить подтверждение.

FR-3.4 При подтверждении `/confirm_schedule` система должна сохранить расписание в базе данных и отправить уведомление участникам группы (если группа есть).

FR-3.5 При обнаружении ошибок или по запросу пользователя `/edit_schedule` система должна позволить отредактировать или загрузить расписание заново.

# FR-4: Импорт заданий

FR-4.1 Система должна предоставить пользователям с ролью староста или индивидуал возможность импортировать задания через команду `/import_homework`.

FR-4.2 Система должна предложить два способа импорта:
* Отправить фото 
* Ввести вручную 

FR-4.3 После подтверждения `/send_to_group` система должна сохранить задания и отправить их участникам группы или индивидуалу.

# FR-5: Управление заданиями

FR-5.1 Система должна предоставить пользователям с любой ролью возможность просмотреть список всех заданий через команду `/list`.

FR-5.2 Система должна отображать список заданий с указанием предмета, названия, дедлайна.

FR-5.3 Система должна предоставить пользователю возможность отметить задание как выполненное через команду `/done [id задания]`.

FR-5.4 Система должна обновить статус задания на "выполнено" и подтвердить действие пользователю.

FR-5.5 Система должна хранить историю выполнения заданий для каждого пользователя индивидуально (даже в рамках одной группы).

# FR-6: Система мемов

FR-6.1 Система должна автоматически выдавать мем при отметке задания как выполненного через команду `/done [id задания]`.

FR-6.2 Мем должен отправляться пользователю в виде изображения или текста.

# FR-7: Настройки напоминаний

FR-7.1 Система должна предоставить пользователям всех ролей возможность настройки напоминаний через команду `/remind`.

FR-7.2 Система должна предложить следующие режимы напоминаний:
* Автоматический (по умолчанию - за 24 часа и за 3 часа до дедлайна).
* Ручной (пользователь устанавливает своё время).
* Отключить напоминания `/remind_off`.
* Включить напоминания `/remind_on` (если раньше были отключены).

FR-7.3 Система должна отправлять напоминания пользователю в соответствии с его сохранёнными настройками.

# FR-8: Мотивационные фразы

**Описание**: Студент получает мотивационные фразы в одном сообщении с дедлайнами заданий.

# FR-9: Уведомления от старосты.

**Описание**: Староста может отправить сообщение, которое бот отправит каждому участнику  данной группы.

# FR-10: Автоматичсекая очистка БД.

**Описание**: БД будет автоматически очищаться каждые пол года.

# FR-11: Свободное время.

**Описание**: Студент в свободное время может запросить в системе 2 задания с близжайшими дедлайнами.

# FR-12: Импорт расписания сессии.

**Описание**: Староста или индивидуал импортирует расписание удобным ему способом (Excel-файл, фото (OCR) или ручной ввод) в систему.

# FR-13: Удаление предмета.

**Описание**: Староста или индивидуал может удалить из расписания предмет, который закончился.

# FR-14: Список всех команд.

**Описание**: Студент может получить список всех команд бота при вооде функции `/commands`.
