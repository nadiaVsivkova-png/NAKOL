# FR-1: Регистрация и роли

**Описание**: Не авторизованный пользователь заходит в ТГ-Бот,  переходит к авторизации и получает доступ к функциям, в зависимости от его роли.

**Предусловие**: Пользователь не авторизован. Бот активен.

**Основной поток**:
1. Пользователь нажимает кнопку "Старт".
2. Система отображает приветственное сообщение и дает выбрать роль:  "Я староста", "Я в группе", "Я сам за себя".
3. Пользователь выбирает роль.
4. Система запомнила роль пользователя.

**Результат**: Пользователь авторизован в системе. Система запомнила роль пользователя и внесла его в базу данных.
# FR-2: Управление группами

**Описание**: Староста создал группу с уникальным кодом доступа. Участники могут получить доступ в группе введя код доступа.

**Предусловие**: Пользователь-староста авторизован в боте. Бот активен. 

**Основной поток**:
1. Староста регистрирует группу.
2. Система вносит группу в базу и генерирует уникальный код доступа.

**Результат**: Староста создал группу с кодом доступа.

# FR-3: Импорт расписания

**Описание**: Пользователь импортирует расписание в систему.

**Предусловие**: Пользователь авторизован как староста или индивидуал. Бот активен.

**Основной поток**:
1. Пользователь вводит команду `/import_schedule` для импорта расписания.
2. Система предлагает 3 способа для импорта: Excel-файл, фото, ручной ввод.
3. Пользователь выбирает один способ.
4. Система, в зависимости от выбранного пользователем способа, запрашивает данные.
5. Пользователь отправляет расписание системе.
6. Система получает, обрабатывает и выводит полученные данные.
7. Система просит перепроверить данные.
8. Пользователь, если все правильно, подтверждает данные.
9. Система сохраняет данные.

**Альтернативный поток**:
1. На 8 этапе **Основного потока** пользователь решает исправить данные.
2. Пользователь вводит команду `/edit_schedule` для изменения данных.
3. Система выводит данные для исправления (в формате, в зависимости  от выбранного пользователем способа импорта расписания).
4. Пользователь исправляет данные и отправляет системе.
5. Система просит перепроверить данные.
6. Пользователь, если все правильно, подтверждает данные.
7. Система сохраняет данные

**Результат**: В системе есть расписание.
# FR-4: Импорт заданий

**Описание**: Пользователь импортирует задания в систему.

**Предусловие**: Пользователь авторизован как староста или индивидуал. Бот активен.

**Основной поток**:
1. Пользователь вводит команду `/import_homework` для импорта заданий.
2. Система предлагает 2 способа: отправить фото, ввести вручную.
3. Пользователь выбирает один способ.
4. Система, в зависимости от выбранного фото, выводит данные для импорта заданий.
5. Пользователь вводит данные.
6. Система обрабатывает и выводит полученные данные для проверки пользователю.
7. Пользователь проверяет данные, если все хорошо, то подтверждает изменения.
8. Система сохраняет данные.

**Альтернативный поток**: 
1. На 7 пункте **основного потока** пользователь решил изменить данные и вводит команду для изменений.
2. Система выводит данные для исправления (в формате, в зависимости  от выбранного пользователем способа импорта заданий).
3. Пользователь исправляет данные и отправляет системе.
4. Система просит перепроверить данные.
5. Пользователь, если все правильно, подтверждает данные.
6. Система сохраняет данные

**Результат**: В системе есть задания.
# FR-5: Управление заданиями

**Описание**: Пользователь может получить список всех заданий и отметить те, которые он выполнил.

**Предусловие**: Пользователь авторизован. Бот активен.

**Основной поток**:
1. Пользователь вводит команду `/list`/, чтобы получить список всех заданий.
2. Система выводит список всех заданий.
3. Пользователь вводит команду `/done ID`, чтобы отметить выполненным конкретное задание.
4. Система убирает из списка задач выполненные задания.

**Результат**: Пользователь вывел список всех задач и отметил сделанные. Система убрала из списка задач выполненные задачи.
# FR-6: Система мемов

**Описание**: Пользователь может получить мем за каждое выполненное задание. Сделал работу и отметил в боте - получил мем. 

**Предусловие**: Пользователь зарегистрирован в боте и ему назначена роль. Бот активен. У пользователя есть задания, которые нужно выполнить.

**Основной поток**: 
1. Пользователь получает список заданий.
2. Пользователь отмечает выполненным задание.
3. Система отправляет мем пользователю.

**Результат**: Пользователь получил мем.
# FR-7: Настройки напоминаний

**Описание**: Авторизованный пользователь изменяет настройки напоминаний о дедлайнах: выбирает режим (автоматический, ручной, отключение) и устанавливает время уведомлений.

**Предусловие**: Пользователь авторизован в системе. Бот активен.

**Основной поток**:
1. Пользователь отправляет команду `/remind`.
2. Система отображает текущие настройки (при регистрации по умолчанию напоминания приходят 2 раза: за день до дедлайна в 18:00 и в 8:00 утром в день дедлайна) и предлагает варианты:  
	   - "Перейти на автоматический режим"  
	   - "Установить своё время" 
		 - "Отключить напоминания" 
	   - "Включить напоминания" 
3. Пользователь выбирает один из вариантов.
4. В зависимости от выбора, система запрашивает дополнительные данные.
5. Пользователь вводит требуемые параметры.
6. Система сохраняет новые настройки и отправляет подтверждение.

**Результат**: Настройки напоминаний изменены в соответствии с выбором пользователя. Система запомнила новые параметры и будет использовать их для отправки уведомлений.

**Основной поток**:
1. Система отправляет пользователю напоминание о заданиях с мотивационной фразой.

**Результат**: Пользователь получил мотивационную фразу.
